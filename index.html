Low-Code ETL · M365

Microsoft Planner Automation Pipeline

Power AutomateSQL ServerPower BIGraph APIDAX
Engineering Challenge

Extract 3,389 tasks from paginated Planner APIs (100-item cap per call), normalize deeply nested JSON, and build a relational SQL schema — all without traditional ETL tooling.

Solution
  • "Do Until" loops with @odata.nextLink — processed 3,389 tasks in 37 minutes
  • Star schema in SQL Server: Fact_Tasks + Dim_Plans, Buckets, Users, Dates
  • DAX time intelligence: Cette Semaine, En Retard, Semaine Prochaine
  • Delta-only incremental loads — 80% reduction in API consumption
